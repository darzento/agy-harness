# omc-code-synthesizer (코드 합성기)

이 스킬은 아키텍트가 정의한 태스크 카드를 기반으로 실제 소스코드를 안전하게 생성하고, 기존 코드와의 병합 및 리팩토링 작업을 자동 수행합니다.

## When to use this skill
- 오케스트레이터로부터 할당된 태스크 카드와 스펙에 따라 신규 코드를 생성해야 할 때 호출합니다.
- 코드 검증 단계에서 오케스트레이터가 코드 리뷰어(`omc-reviewer`)의 피드백을 담아 코드 리팩토링 및 에러 수정을 요청(`REFACTOR_REQUEST`)했을 때 호출합니다.

## Instructions
1. **스펙 준수 구현**:
   - `tasks.json` 혹은 전달받은 태스크 스펙에 기술된 모듈 사양, API 구조, 인터페이스 규격을 일관되게 적용하십시오.
2. **안전한 작업 격리**:
   - 생성하거나 수정한 모든 코드는 1차적으로 [_workspace/](file:///D:/01_Source/oh-my-agy/_workspace) 하위에 저장되도록 작성하여 메인 소스 트리의 오염을 방지합니다.
3. **효율적 코드 패치**:
   - 기존 파일 수정 시, 전체 덮어쓰기(`write_to_file`)보다는 `replace_file_content` 또는 `multi_replace_file_content` 도구를 사용해 변경이 일어난 범위만 명확하게 편집하십시오.
4. **리팩토링 요청 처리**:
   - 수신한 피드백에서 오류 원인과 제안 사항을 파악하고, 디버깅을 위해 수정해야 하는 대상 파일을 정밀 분석하여 코드를 수선하십시오.

## Workflow
1. **태스크 수신 및 대조**: 할당된 작업 카드 내용과 기존 파일의 위치를 확인합니다.
2. **코드 생성 및 패치**: 격리 경로인 `_workspace/` 하위에 코드를 안전하게 구현하거나, 기존 파일의 사본을 바탕으로 패치 코드를 적용합니다.
3. **코드 자체 검토**: 구현된 코드의 예외 처리, 타입 무결성 등을 자체적으로 확인합니다.
4. **작업 완료 통지**: 수정한 파일 절대 경로를 포함한 JSON 규격 메시지를 오케스트레이터에게 전송하여 다음 단계인 QA 검증을 트리거합니다.
