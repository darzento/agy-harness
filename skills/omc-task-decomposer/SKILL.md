# omc-task-decomposer (태스크 분할기)

이 스킬은 복잡한 소프트웨어 요구사항이나 설계 목표를 분석하여, 다중 에이전트 환경에서 충돌 없이 병렬로 구현할 수 있는 최적의 작업 맵을 기획하고 구조화합니다.

## When to use this skill
- 오케스트레이터로부터 새로운 개발 구현 지시나 복잡한 리팩토링 요청(`REQUEST_REVIEW` 또는 `STATUS_UPDATE`)을 전달받아 시스템 아키텍처 기획이 필요한 상황에 사용합니다.
- 기존 소스코드베이스의 종속 관계와 의존성을 사전에 분석하여 기능별 병렬 작업 단위를 설정할 때 사용합니다.

## Instructions
1. **코드 및 컨텍스트 리서치**:
   - `grep_search` 및 `read_file`을 이용하여 현재 프로젝트 내부의 소스코드 구조, API 스펙, 설정 파일 등을 먼저 조사하십시오.
2. **JSON 기반 태스크 명세 스캐폴딩**:
   - 분할된 각 태스크는 ID, 제목, 상세 명세(구현할 함수/인터페이스 스펙), 선행되어야 할 태스크 ID 목록(의존성), 그리고 작업 대상 파일 경로가 포함되어야 합니다.
   - 결과 파일은 반드시 [_workspace/tasks.json](file:///D:/01_Source/oh-my-agy/_workspace/tasks.json) 파일로 기록하십시오.
3. **결과 보고**:
   - 작성이 완료되면 오케스트레이터에게 JSON 페이로드 규격으로 통지하며, `target_artifact`에 `_workspace/tasks.json` 절대 경로를 기입하십시오.

## Workflow
1. **요구사항 파악**: 오케스트레이터의 기획 오더를 분석하고, 관련 있는 기존 코드를 검사합니다.
2. **종속성 트리 설계**: 신규 개발 모듈과 기존 모듈 간의 임포트 및 상호작용 관계를 파악하여 선후 관계를 구성합니다.
3. **태스크 파일 저장**: 구조화된 JSON 데이터(`tasks.json`)를 `_workspace/` 디렉토리에 저장합니다.
4. **완료 송신**: `send_message`를 사용해 오케스트레이터에게 태스크 분할 작업이 완수되었음을 통보하고 리뷰를 요청합니다.
