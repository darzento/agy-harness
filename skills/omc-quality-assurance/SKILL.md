# omc-quality-assurance (품질 검증기)

이 스킬은 작성된 코드가 사전에 정의된 소프트웨어 스펙을 충족하는지, 컴파일/인터프리터 빌드 상의 에러는 없는지, 보안상 취약점이나 메모리 누수 가능성이 없는지 자동 검사하고 리포트합니다.

## When to use this skill
- 오케스트레이터로부터 구현이 끝난 모듈의 검증 오더(`REQUEST_REVIEW`)를 받았을 때 사용합니다.
- 변경된 코드가 시스템 전체의 통합에 무리가 없는지 확인하기 위해 테스트 세트를 빌드하고 실행해야 할 때 사용합니다.

## Instructions
1. **코드 및 정적 분석**:
   - `_workspace/` 하위에 작성된 코드를 로드하여 구문 오류(Syntax), 안티패턴, 린트 오류를 1차 분석하십시오.
2. **테스트 스니펫 생성 및 수행**:
   - 코드가 제대로 기능하는지 검증하기 위한 간단한 단위 테스트 코드를 `_workspace/` 하위에 작성하고, `command` 도구를 실행해 테스트를 수행하십시오.
3. **엄격한 결과 판정**:
   - 검증 결과는 `PASS` 또는 `FAIL`로 명확하게 판정해야 합니다.
   - 테스트 실패, 린트 위반, 혹은 명세 누락이 1개라도 발생한 경우 판정은 반드시 `FAIL`이어야 하며, 구체적인 수정 가이드를 피드백 본문에 채워야 합니다.
4. **결과 리포팅**:
   - 최종 검증 결과와 분석 리포트를 JSON 메시지 형태로 조립하여 오케스트레이터에게 즉시 송신하십시오.

## Workflow
1. **검증 모듈 확인**: 오케스트레이터가 보고한 검증 대상 파일의 코드를 읽어 들입니다.
2. **정적/동적 검증**: 린터를 수행하거나, 자율적으로 테스트 도구를 호출하여 결과 로그를 수집합니다.
3. **품질 검토 리포트 작성**: 검사 결과 발생한 경고 및 에러 내용을 요약한 보고서 파일을 `_workspace/` 하위에 기록합니다.
4. **판정 메시지 송신**: PASS/FAIL 판정값과 함께 상세 개선 방향이 담긴 JSON 메시지를 오케스트레이터에게 회신합니다.
